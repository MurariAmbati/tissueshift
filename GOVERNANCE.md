# TissueShift Governance

## Current Model: Benevolent Dictator

TissueShift is currently maintained by its original author. All architectural decisions, merge approvals, and release decisions go through the maintainer.

## Transition Plan

When TissueShift reaches **10+ regular contributors** (defined as 3+ merged PRs over 3+ months), governance will transition to a **Steering Committee** model:

- **Steering Committee**: 3-5 members elected from active contributors
- **Responsibilities**: Architectural decisions, release planning, conflict resolution
- **Terms**: 1 year, renewable
- **Voting**: Simple majority for routine decisions, 2/3 for architectural changes

## Decision-Making Areas

| Area | Current | Future |
|------|---------|--------|
| Code merges | Maintainer | Any committee member |
| Architecture changes | Maintainer | Committee vote (2/3) |
| New benchmark tracks | Maintainer | Committee vote (majority) |
| New data sources | Maintainer | Committee vote (majority) |
| Release versioning | Maintainer | Committee vote (majority) |
| Ethics/safety issues | Maintainer | Full committee required |

## Contributor Roles

- **Contributor**: Anyone with a merged PR
- **Regular Contributor**: 3+ merged PRs over 3+ months
- **Maintainer**: Full write access, release authority
- **Committee Member**: Architectural decision-making authority (future)
